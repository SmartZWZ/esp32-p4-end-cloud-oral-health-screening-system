# 基于 ESP32-P4 的端云协同口腔健康筛查系统

本仓库用于整理一个 AIoT 口腔健康筛查项目：端侧使用 ESP32-P4 采集口腔图像并提供实时交互，云端使用 AI 模型识别疑似病灶区域，最终在屏幕或 Web 端叠加显示筛查结果。

> 项目定位：口腔健康辅助筛查与风险提示，不作为医学诊断结论。

## 当前目标

先完成一个可演示闭环：

1. ESP32-P4 或模拟设备采集口腔图像。
2. 端侧完成图像质量检查、基础提示和轻量粗筛。
3. 合格图像上传到云端推理服务。
4. 云端识别疑似病灶区域并返回框/掩膜/置信度。
5. 屏幕或 Web 端叠加显示“病灶在哪里”。

## 端云模型分工

- ESP32-P4 端侧模型：轻量检测模型，用于离线粗筛、图像质量提示和疑似区域提示。
- 云端主模型：病灶检测/分割模型，用于更准确地定位疑似龋齿、龋洞、裂纹等区域。
- 云端部署：优先使用本地服务器 Intel Arc A380 + OpenVINO GPU；阿里云 2c2g 更适合作为 API 网关、备用低频推理或远程管理入口。

## 当前 AI 实验结论

已有公开/整理数据集可转为 YOLO 格式，但端侧 `yolo11n` 检测模型在 `imgsz=320` 上效果有限：

- 训练集：1991 张图
- 验证集：254 张图
- 测试集：250 张图
- 病灶框总数：6169
- `yolo11n detect imgsz=320` 最好 `mAP50` 约 0.23

初步判断：病灶目标小、形态差异大，且数据域与真实口腔摄像头画面可能存在差异。后续应优先补充更贴近真实设备采集条件的数据。

## 协作建议

1. 不提交大模型权重、原始数据集和训练输出，统一放到网盘、对象存储或本地路径。
2. 公共接口先写到 `shared/api-contracts/` 或 `shared/schemas/`。
3. 每次提交尽量只包含一个明确任务，例如 `ai: add edge dataset converter`。
4. 文档、代码和实验记录同步更新，方便后续复现。
