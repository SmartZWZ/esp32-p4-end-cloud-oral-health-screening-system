# 基于 ESP32-P4 的端云协同口腔健康筛查系统

本仓库采用单仓库（monorepo）结构，统一管理 ESP32-P4 固件、云端后端、Web 前端、AI 模型实验、接口协议、文档和演示资源。

## 目录结构

```text
.
├─ firmware/   ESP32-P4 固件工程
├─ backend/    云端后端服务
├─ web/        Web 前端
├─ ai/         模型训练、推理和数据实验
├─ shared/     端云接口协议、JSON Schema、示例数据
├─ docs/       计划书、系统设计、接口文档、答辩材料
├─ scripts/    开发、部署、数据处理脚本
└─ assets/     图片、演示素材和原型资源
```

## 推荐分工

- 固件端：负责摄像头、ToF、IMU、LVGL、触摸屏交互、语音指令、图像质量检测和上传。
- 后端：负责用户、设备、家庭成员、报告、图片上传、模型调用和数据隔离。
- 前端：负责登录、设备绑定、成员切换、报告查看、历史记录和复查提醒。
- AI：负责图像质量评估、牙齿区域分割、风险区域检测、推理服务和实验记录。
- 文档：负责需求分析、系统设计、接口文档、测试记录、答辩材料。

## 团队协作建议

1. 每个成员从 `main` 新建自己的功能分支，例如 `feature/web-login`、`feature/firmware-camera-preview`。
2. 公共接口先写到 `shared/api-contracts/` 或 `shared/schemas/`，再分别实现固件、后端和前端。
3. 大模型、训练数据、导出的模型权重尽量不要直接提交到 Git，先放说明文件或使用外部网盘/对象存储。
4. 每次提交尽量只包含一个清晰任务，提交信息写明模块和改动，例如 `backend: add report list api`。

## 当前阶段目标

先完成一个可演示闭环：

1. ESP32-P4 或模拟设备采集/选择口腔图像。
2. 本地完成图像质量判断和基础提示。
3. 上传合格图像到云端。
4. 云端完成风险标注和报告生成。
5. Web 端查看本次报告和历史记录。
