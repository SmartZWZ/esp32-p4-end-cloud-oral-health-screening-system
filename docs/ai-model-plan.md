# AI 模型方案与实验记录

## 模型分工

### 端侧模型

- 任务：检测
- 类别：`lesion`
- 来源：将 `Caries/Cavity/Crack` 合并成一个类别
- 输入尺寸：优先尝试 320，必要时训练 640 后再压缩
- 部署目标：ESP32-P4 端侧轻量粗筛

### 云端模型

- 任务：检测或分割
- 类别：可先使用 `lesion`，后续再拆分为 `Caries/Cavity/Crack`
- 输入尺寸：512 或 640
- 部署目标：Intel Arc A380 + OpenVINO GPU

## 当前训练结果

`yolo11n detect imgsz=320` 训练约 52 个 epoch 后，最好结果：

```text
mAP50:    0.23369
mAP50-95: 0.09033
precision: 0.38421
recall:    0.27592
```

初步判断：

- 320 输入下，小病灶目标过小，模型召回不足。
- 病灶形态差异大，单类 lesion 仍然不好学。
- 公开数据与真实口腔摄像头画面可能存在数据域差异。

## 下一步建议

1. 训练 `imgsz=640` 的诊断性模型，判断数据本身是否可学。
2. 关闭强增强，降低小病灶被破坏的概率。
3. 补充真实设备采集数据，优先采集清晰、光照稳定、有病灶标注的样本。
4. 训练完成后导出 OpenVINO，在 Arc A380 上做 FPS benchmark。
