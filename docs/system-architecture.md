# 系统架构说明

## 目标

构建一个端云协同的口腔健康筛查系统：

- 端侧使用 ESP32-P4 采集口腔图像、实时预览并给出拍摄质量提示。
- 云端使用 AI 模型定位疑似病灶区域。
- 屏幕或 Web 端叠加显示检测框/掩膜和风险提示。

## 推荐架构

```text
ESP32-P4 设备
  ├── 摄像头采集
  ├── LED 补光
  ├── ToF/IMU 辅助拍摄质量判断
  ├── 本地轻量 lesion 检测
  └── 上传关键帧

云端/本地服务器
  ├── 图像接收 API
  ├── OpenVINO GPU 推理服务
  ├── 结果后处理
  ├── 报告生成
  └── 历史数据管理

Web/屏幕显示
  ├── 实时画面
  ├── 病灶框/掩膜叠加
  ├── 置信度和风险提示
  └── 报告查询
```

## 服务器算力

南昌本地服务器已验证：

- CPU：Intel Xeon E5-2673 v3，12 核 24 线程
- GPU：Intel Arc A380，约 6GB 显存
- OpenVINO：可识别 `GPU Intel Arc A380 Graphics (dGPU)`
- FFmpeg：VAAPI/QSV 可用，支持 H.264/HEVC/VP9/AV1 硬件编解码

建议：

- 显示链路保持 1080P 30FPS。
- AI 推理输入使用 512 或 640，再映射回 1080P。
- Arc A380 优先用于 OpenVINO 推理，而不是主训练。
